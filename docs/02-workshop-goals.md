# Workshop Goals

TeamPulse teaches Angular through a complete but compact business application. The goal is to show how Angular concepts fit together in an app that looks and behaves like something engineers might maintain at work.

## Angular Learning Goals

- Standalone components.
- Feature-based organization.
- Routing and lazy-loaded route components.
- Route guards.
- Role-based rendering.
- Services and dependency injection.
- HttpClient and typed API services.
- HTTP interceptors.
- Signals and computed state.
- Reactive Forms and validators.
- Pipes and directives.
- Component inputs and outputs.
- Animations.
- LocalStorage-backed state.
- Loading, empty, and error states.

## PrimeNG Learning Goals

- Use PrimeNG controls in real page workflows.
- Build tables with pagination, sorting, and filtering.
- Use dialogs for create/edit forms.
- Use tags, cards, progress bars, charts, date pickers, sliders, selects, and buttons.
- Keep PrimeNG visually consistent with a custom design system.
- Support dark and light themes.

## Backend Teaching Goal

The backend is intentionally simple. It provides real HTTP endpoints and realistic seeded data, but avoids database, EF Core, Identity, and JWT complexity. This lets the frontend demonstrate real integration while keeping workshop time focused on Angular.
